# Candidate Search

AI-подбор кандидатов: вы описываете требования и ключевые слова, приложение ищет
кандидатов по нескольким источникам и ранжирует их с помощью **LLM**
(Claude или Google Gemini — по наличию ключа).

> **Важно про «поиск по всему интернету».** Индекса всего интернета нет ни у
> кого, кроме поисковиков. Поэтому «поиск кандидатов» здесь — это поиск по
> **конкретным подключаемым источникам**: GitHub (официальный API) и Web
> (X-ray-поиск публичных страниц через Serper.dev/SerpAPI). Источники —
> расширяемые (см. `server/src/search/providers`).

> **Можно собрать полностью бесплатно, без карты:** хостинг Render (free) +
> источник GitHub (без ключа) + ранжирование Google Gemini (free-tier ключ в
> [AI Studio](https://aistudio.google.com/apikey)) + опционально веб-поиск
> Serper.dev (2500 запросов free). Платные Claude/SerpAPI — по желанию.

---

## Архитектура

```
[ Angular client ]  ──POST /api/search──▶  [ NestJS server ]
   форма запроса                              ├─ GithubProvider   (GitHub Search API, бесплатно)
   карточки кандидатов                        ├─ WebProvider      (X-ray: Serper.dev / SerpAPI)
   скор + объяснение                          └─ RankingService   (Claude / Gemini: скор 0-100 + объяснение)
```

- **`server/`** — NestJS API. Провайдеры источников за общим интерфейсом
  (`SourceProvider`), ранжирование выбирает LLM по наличию ключа (Claude → Gemini
  → keyword-фолбэк). Всё деградирует «мягко» и **без моков**: нет ключа источника
  → источник пропускается (пустой результат); нет ключа LLM → скоринг по ключевым
  словам. Один упавший источник не ломает весь поиск. В продакшене NestJS
  дополнительно раздаёт собранный Angular (single-service), поэтому весь продукт
  живёт на одном URL.
- **`client/`** — Angular 21 (standalone + signals). Форма требований/ключевых
  слов/локации/источников и список карточек кандидатов со скором релевантности
  и объяснением от LLM.

## Быстрый старт

```bash
# 1. Установить зависимости обоих пакетов
npm run install:all

# 2. Настроить ключи бэкенда (все опциональны — без них работают фолбэки)
cp server/.env.example server/.env
#   ANTHROPIC_API_KEY — ранжирование через Claude
#   SERPAPI_KEY       — реальный X-ray-поиск (иначе demo-результаты)
#   GITHUB_TOKEN      — выше лимиты GitHub (поиск работает и без токена)

# 3. Запустить (в двух терминалах)
npm run dev:server   # http://localhost:3000
npm run dev:client   # http://localhost:4200
```

Открыть <http://localhost:4200>, ввести требования и нажать «Найти кандидатов».

## Деплой на Render (один публичный URL)

В проде один web-сервис отдаёт и API, и Angular. Конфигурация —
[`render.yaml`](./render.yaml) (Blueprint).

1. Запушить репозиторий (уже сделано).
2. Render → **New → Blueprint** → выбрать этот репозиторий → **Apply**.
   Render прочитает `render.yaml`, соберёт (`npm run render-build`) и запустит
   (`npm run render-start`).
3. В настройках сервиса вставить секреты (они помечены `sync: false`, в git не
   попадают). Для бесплатного варианта достаточно:
   - `GEMINI_API_KEY` — ранжирование (free, без карты);
   - `SERPER_API_KEY` — веб-поиск (опционально; GitHub-источник работает и без него).

   Платные альтернативы (если предпочитаете): `ANTHROPIC_API_KEY`, `SERPAPI_KEY`.
   `GITHUB_TOKEN` — опционально, для лимитов.
4. После деплоя Render выдаёт публичный `https://<name>.onrender.com` — этот
   адрес открывается и с телефона.

> На free-плане сервис засыпает при простое: первый запрос после паузы может
> подниматься ~30–50 секунд.

## API

`POST /api/search`

```jsonc
{
  "requirements": "Senior Angular developer, RxJS/NgRx, 5+ years, Berlin",
  "keywords": ["Angular", "RxJS", "TypeScript"],
  "location": "Berlin",          // опционально
  "sources": ["github", "web"],  // какие источники опрашивать
  "limit": 15
}
```

Ответ: `{ count, rankedBy: "claude" | "gemini" | "keywords", candidates: [...] }`,
где у каждого кандидата есть `score` (0–100) и `reasoning` (объяснение).

## Переменные окружения (`server/.env`)

Для ранжирования и веб-поиска задаётся **по одному ключу** на функцию — приложение
берёт тот, что найдёт.

| Переменная          | Обязательна | Назначение                                                             |
| ------------------- | ----------- | ---------------------------------------------------------------------- |
| `PORT`              | нет         | Порт API (по умолчанию 3000)                                          |
| `CORS_ORIGIN`       | нет         | Разрешённые origin фронтенда (по умолчанию `http://localhost:4200`)   |
| `GEMINI_API_KEY`    | нет\*       | Ранжирование через Gemini (**free, без карты**)                       |
| `GEMINI_MODEL`      | нет         | Модель Gemini (по умолчанию `gemini-1.5-flash`)                       |
| `ANTHROPIC_API_KEY` | нет\*       | Ранжирование через Claude (приоритетнее Gemini, если задан)           |
| `ANTHROPIC_MODEL`   | нет         | Модель Claude (по умолчанию `claude-haiku-4-5-20251001`)              |
| `SERPER_API_KEY`    | нет\*       | X-ray-поиск через Serper.dev (**free 2500 запросов**)                |
| `SERPAPI_KEY`       | нет\*       | X-ray-поиск через SerpAPI (альтернатива Serper)                       |
| `GITHUB_TOKEN`      | нет         | Поднимает лимиты GitHub Search API                                    |

\* Не обязательны для запуска. Без LLM-ключа — скоринг по ключевым словам; без
веб-ключа — источник Web пропускается (GitHub-источник работает без ключей).

## Правовые заметки про X-ray-поиск

X-ray-поиск возвращает публично проиндексированные страницы. Массовый сбор
данных с площадок вроде **LinkedIn нарушает их Terms of Service** — поэтому
`WebProvider` по умолчанию таргетит X-ray-дружелюбные площадки (личные сайты на
`github.io`, Stack Overflow, dev.to, GitLab, Medium). Список сайтов и
compliance-политику настраивайте в `server/src/search/providers/web.provider.ts`.

## Заметка про песочницу

В изолированной среде разработки прямые запросы к `api.github.com` могут
блокироваться прокси (сессия ограничена своими репозиториями). Это ограничение
окружения, а не приложения: `GithubProvider` в этом случае корректно возвращает
пустой результат. В обычном деплое GitHub-поиск работает без ограничений.
